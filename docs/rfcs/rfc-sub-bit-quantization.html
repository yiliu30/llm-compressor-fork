RFC: Sub-Bit Quantization (1–8 bit) End-to-End Support

Author: Yi Liu  |  Date: 2026-05-20  |  Status: Draft

1. Summary

Enable arbitrary integer bit-width (1–8) weight quantization through the full pipeline: llm-compressor → compressed-tensors → vLLM → humming kernel. Two-stage rollout: Stage 1 requires explicit --quantization humming; Stage 2 makes humming auto-selected for sub-bit via the standard compressed-tensors path.

2. Motivation

  • Humming kernels already support INT 1–8 bit GEMM with full MMA optimization.
  • The upstream toolchain (compressed-tensors, llm-compressor) artificially restricts to 4/8-bit.
  • No existing vLLM kernel (Marlin, Machete, Exllama) supports sub-bit — humming is the only option.
  • Sub-bit (especially 3-bit) offers better accuracy/size tradeoff than 2-bit or 4-bit for many models.

5.1 New: HummingMPLinearKernel

# vllm/model_executor/kernels/linear/mixed_precision/humming_mp.py

class HummingMPLinearKernel(MPLinearKernel):

    @classmethod
    def get_min_capability(cls) -> int:
        return 75  # humming supports sm75+

    @classmethod
    def can_implement(cls, config: MPLinearLayerConfig) -> tuple[bool, str|None]:
        # Accept any INT weight type with 1-8 bits
        if config.weight_type.is_integer() and 1 <= config.weight_type.size_bits <= 8:

Priority placement rationale: Humming goes after Marlin so that for 4/8-bit, proven kernels are preferred. For 3/5/6/7-bit, Marlin/Machete/AllSpark all return can_implement=False, so humming auto-wins. If benchmarks show humming is faster than Marlin at 4-bit, promote it above Marlin later.

6. UX Changes

6.1 Compression (llm-compressor)

# Before (only 4/8 worked):
recipe = GPTQModifier(scheme="W4A16")

# After Stage 1 — any bit-width, multiple algorithms:
recipe = AutoRoundModifier(scheme="W3A16")     # AutoRound 3-bit (recommended first)
recipe = GPTQModifier(scheme="W3A16")          # GPTQ 3-bit symmetric, group=128
recipe = GPTQModifier(scheme="W5A16")          # GPTQ 5-bit symmetric, group=128
recipe = QuantizationModifier(scheme="W3A16")  # RTN 3-bit (fastest, lower quality)

# Combine with transforms (AWQ scaling + 3-bit quantization):
recipe = [
    AWQModifier(targets=["Linear"]),
    GPTQModifier(scheme="W3A16"),
]

Output: a standard HuggingFace checkpoint with quantization_config.quantization_method = "compressed-tensors" and num_bits=3 in the config groups.

6.2 Inference (vLLM)

StageCommandBehavior
S1 vllm serve ./my-3bit-model --quantization humming Explicit humming flag required. vLLM loads via HummingConfig path, repacks weights, runs humming kernel.
S2 vllm serve ./my-3bit-model Auto-detected. vLLM reads compressed-tensors config, CompressedTensorsWNA16 scheme selects HummingMPLinearKernel automatically (only kernel that supports sub-bit). No flag needed.
Backward compatibility: Existing 4-bit and 8-bit models are unaffected in both stages. Marlin/Machete remain the default kernels for those bit-widths. The --quantization humming flag continues to work as an explicit override for any bit-width.
